Objetivo

Desenvolver um dashboard inovador e interativo utilizando HTML, CSS, JavaScript (utilizando as bibliotecas Echarts e Bootstrap 5) para visualizar e analisar dados de saúde pública do Estado de Goiás. O dashboard deve proporcionar insights valiosos e facilitar a tomada de decisões pelos gestores de saúde, além de oferecer importantes análises para a sociedade.

Na plataforma Dados abertos existe um conjunto de dados que podem ser acessados de forma pública, separados em diferentes temas. Escolha um tema, e desenvolva um dashboard sobre. Na qual serão avaliados aspectos como Criatividade, Design e Capacidade Analítica.

Nessa plataforma é possível consumir esses dados via API. Na qual, existem instruções de como esses dados podem ser consultados utilizando requisições HTTP para desenvolver esse painel. Por exemplo, para qualquer conjunto de dados basta clicar no botão "API de dados" https://dados.saude.go.gov.br/dataset/covid-19/resource/794598c6-cdf8-4d70-9183-d069004d09c7

Requisitos Técnicos
  • HTML e CSS: Estruturar e estilizar o dashboard de forma responsiva e intuitiva utilizando HTML5 e CSS3.
  • JavaScript: Implementar a lógica de interatividade e manipulação dos dados utilizando JavaScript.
  • ECharts: Utilizar a biblioteca ECharts para criar gráficos interativos que permitam a visualização clara dos dados.
  • Bootstrap 5: Utilizar Bootstrap 5 para garantir a responsividade e um design moderno ao dashboard.
Funcionalidades Esperadas
Visualização de Gráficos
  • Gráficos para representar diferentes métricas e tendências ao longo do tempo.
  • O painel deve conter pelo menos 4 gráficos (os tipos de gráficos são da escolha do analista de BI).
Filtros e Seleções
  • Filtros para selecionar dados por período, região, tipo de doença, etc.
  • O painel deve conter pelo menos 1 filtro.
Critérios de Avaliação
  • Inovação e Criatividade: O uso criativo das ferramentas para fornecer insights valiosos.
  • Usabilidade e Design: A intuitividade, clareza e estética do dashboard.
  • Interatividade e Funcionalidade: A eficácia dos gráficos interativos e a utilidade dos filtros e seleções.
  • Organização e Estrutura do Código: A qualidade e organização do código HTML, CSS e JavaScript.
  • Aderência aos Requisitos: Cumprimento dos requisitos técnicos e das funcionalidades esperadas.

O candidato terá um prazo de 72 horas para desenvolver o dashboard e apresentar uma demonstração funcional junto com a documentação técnica do projeto.
